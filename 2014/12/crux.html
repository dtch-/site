Crux as a daily driver

Crux is a source-based linux distro. All utilities must be compiled, including the kernel. While it may seem daunting for someone who's unfamiliar with the process, it's actually quite simple. The general crux workflow is as follows:

  1. Install package(pkg) x
  2. Pkg x isn't ported
  3. Create a port
  4. Run the program
  5. refer to step 1
Creating a port is fairly simple if you have the proper utilities. Z3bra has created `prtmk` for use within crux. You can view it in action here. As well, here is the pkgfile template needed for use with prtmk.

Why use crux?

I used crux for a little over a week before being upset with it. First, let me just say that I absolutely understand the usability and need for a source-based distro. Crux is simple to use, powerful in the right hands, and raw. You make crux with your bare hands.

That being said, I had problems with crux. Mainly due to some packages not wanting to work, even though they work perfectly on Arch. For example, alsa-utils and mpd. MPD would not start, spewing errors of missing config files despite them being there. Alsa would simply state that there was no such file or directory when running `alsamixer`, which I found no fix for. Personally, the bad outweighed the good in crux for me.

My experience

As I said, the bad outweighed the good. I ultimately retired back to Arch. However, as I used crux I found new respect for source-based distros. Using `prtmk` and `prt-get` felt good, natural, as though it was how a distro was meant to be. If I had an extra computer lying about, or just more time in my day, I would definitely install crux again and try to get it working right.

Just try it

Don't simply take my word. If you enjoy distro-hopping and messing with everything, install crux on a secondary computer or in a VM. It's really a great distro. There is a handbook that literally walks you through the entire process of getting and burning the ISO to compiling the kernel and nearly gets you to X.

In closing

I would give crux a 7/10 (whereas Arch receives 8/10) simply due to the problems I've had with it. If I'd had no problems, it would definitely be a 9/10 for me. If you enjoy source-based distros, or want to be a programmer (and want a matching distro), give it a shot. Just make sure to get the new kernel, as crux 3.1 (current version as of writing) ships with 3.12.24 and the latest is 3.17.6 (stable).
